# KEEP ON FRONTIER

An example of the shape, not a list to adopt. Nothing goes on yours because it's on mine.

```
code-review    best candidate passed 22/40 vs baseline 31/40 (71%), n=40,
               CIs 40-69% vs 62-88%. Failure mode: a missed bug reaches
               production. Re-test when a new candidate appears.

send-email     never scored. Irreversible action, ships without review.
               Not a measurement decision. Do not route.
```

Two shapes, and the difference has to be visible on the page. The first entry came off a
scorecard and carries a number, so a better model is a legitimate reason to re-open it. The
second is a policy call made before any measurement, and no benchmark result should move it.

In six months somebody will ask whether this list is still accurate. Those two entries have
completely different answers.
